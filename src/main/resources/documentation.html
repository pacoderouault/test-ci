Introduction

NGSDiag est un outil d'aide à l'interprétation de variants génétiques (SNP/Indels/CNVs).

Il peut importer des fichiers VCF annotés grace à Ensembl VEP.

Gestion des utilisateurs

Le module de gestion des utilisateurs est accessible via le menu Administrateur > Gestion des utilisateurs. Seuls les utilisateurs ayant la permission de gérer les autres utilisateurs ont accès à ce menu.

Groupes d'utilisateurs

Chaque utilisateur appartient à un ou plusieurs groupes. Ces groupes contiennent une liste de permissions, c'est-à-dire des actions à pouvoir réaliser au sein de NGSDiag.

Deux groupes sont initialement créés :

  • ADMIN : possède tous les droits
  • GUEST : ne possède aucun droit (accès en lecture seulement)

Ces deux groupes ne peuvent pas être supprimés.

L'ajout d'un nouveau groupe se fait en cliquant sur le bouton Ajouter un groupe et en lui donnant un nom unique dans la fenêtre qui apparaît.

Pour définir les permissions de ce groupe, sélectionnez le groupe dans le tableau Liste des groupes et choissez ensuite les permissions qui lui seront associées.

Ajouter/Editer un utilisateur

Lors de la première initialisation de NGSDiag, un utilisateur administrateur est créé. (login/mot de passe : admin/admin

Il est très fortement recommandé de désactiver cet utilisateur ou de changer son mot de passe

Pour ajouter un nouvel utilisateur, cliquez sur le bouton Ajouter un utilisateur. Une fenêtre apparaît. Renseignez l'identifiant, qui doit être unique, et son mot de passe. Choisir au moins un groupe auquel rattacher le nouvel utilisateur.

Il est possible de définir une date d'expiration à chaque compte utilisateur. Sélectionnez l'option Définir une date d'expiration et choisissez ensuite la date voulue.

La date d'expiration peut être modifiée en cliquant sur le bouton Modifier la date d'expiration dans le tableau Gestion des comptes utilisateurs.

Le mot de passe d'un utilisateur peut être réinitialisé en cliquant sur l'icône Réinitialiser le mot de passe dans le tableau Gestion des comptes utilisateurs.

Un utilisateur peut être désactivé, c'est-à-dire qu'il lui sera impossible de se connecter avec son compte, mais il ne sera pas supprimé de la base de données. À tout moment un compte désactivé peut être réactivé.

Les groupes de chaques utilisateurs peuvent aussi être modifiés à tout moment.

Paramètres d'analyse

Chaque analyse qui sera importée devra être rattachée à un ensemble de paramètres. Le module de gestion de ces paramètres est accessible via le menu Administrateur > Gestion des paramètres d'analyse.

Panels

Le panel correspond aux régions ciblées lors de la préparation de la librairie (amplicon ou capture). A partir du module de gestion des paramètres d'analyses, sélectionnez l'onglet Gestion des panels. Pour ajouter un panel cliquez sur le bouton Ajouter un panel. Dans la fenêtre qui s'ouvre, renseignez le nom du panel à rajouter (le nom doit être unique) et cliquez sur le bouton Ouvrir un fichier bed pour sélectionner le fichier "bed" qui contient les régions séquencées.

C'est un fichier qui doit contenir au moins les 4 colonnes suivantes :

Contig
Nom du chromosome
Start
Position du début de la région
End
Position de fin de la région
Name
Nom de la région, par exemple le gène ciblé. Ce nom ne doit pas forcément être unique à chaque région.

Exemple :
chr1 6557013 6557096 PLEKHG5
chr1 6557374 6557388 PLEKHG5
chr1 6579499 6579576 PLEKHG5
chr1 10292381 10292497 KIF1B
chr1 10316299 10316386 KIF1B
chr1 10318545 10318735 KIF1B

Une fois un panel ajouté à NGSDiag, il peut être désactivé à partir du tableau Liste des panels. La désactivation d'un panel rendra son utilisation impossible lors de l'ajout d'analyses. Un panel désactivé peut être réactivé à tout moment.

Gènes et transcripts

Les gènes et transcripts correspondent aux noms des gènes et aux transcripts à utiliser pour l'annotation des variants. A partir du module de gestion des paramètres d'analyses, sélectionnez l'onglet Gènes et Transcripts. Pour ajouter un ensemble de gènes et leurs transcripts associés, cliquez sur le bouton Ajouter des gènes/transcripts. Dans la fenêtre qui s'ouvre, renseignez le nom de cet ensemble de gène et transcripts à rajouter (le nom doit être unique) et cliquez sur le bouton Ouvrir le fichier pour charger les gènes et leurs transcripts associés.

C'est un fichier qui doit contenir les 2 colonnes suivantes :

Gene
Nom du gène
Transcripts
Liste des transcripts à utiliser. Il peut y avoir plusieurs transcripts pour un même gène. Il faut séparer leurs identifiants par le caractère : ;

Exemple :
ALPL NM_000478
ANK3 NM_001204403;NM_001204404;NM_020987;NM_001149
AP2S1 NM_001301078;NM_001301076;NM_001301081
ATL1 NM_001127713

Si un gène possède plusieurs transcripts associés, il est possible de choisir le transcript à utiliser en priorité en cliquant dessus. C'est ce transcript qui sera utilisé par défaut pour l'affichage des variants du gène. Par defaut, si un gène ne possède qu'un seul transcript, il est automatiquement sélectionné comme transcript préféré.

Une fois un ensemble de gènes et transcripts ajouté à NGSDiagLim, il peut être désactivé à partir du tableau Liste des gènes/transcripts. La désactivation rendra son utilisation impossible lors de l'ajout d'analyses. Il peut être réactivé à tout moment.

Hotspots

Chaque analyse peut être associée à un ensemble de variants hotspots. Ce sont une liste de variants ou de positions qui seront systématiquement gardées lors de l'importation d'un VCF. Aucun filtre ne sera appliqué sur ces variants.

Pour rajouter des hotspots, à partir du module de gestion des paramètres d'analyses, sélectionnez l'onglet Hotspots. Cliquez ensuite sur le bouton Ajouter des Hotspots. Dans la fenêtre qui s'ouvre, renseignez le nom de l'ensemble de hotspots à rajouter (le nom doit être unique) et cliquez sur le bouton Ouvrir le fichier pour sélectionner le fichier qui contient les hotspots à importer.

C'est un fichier qui doit contenir les 9 colonnes suivantes :

ID
Identifiant du hotspot (par exemple son numéro cosmic)
Contig
Chromosome du hotspot
Start
Position de début du hotspot
End
Position de fin du hotspot
Ref
Allèle de référence
Alt
Allèle alternatif
Gene
Nom du gène
hgvsc
Nomenclature hgvsc
hgvsp
Nomenclature hgvsp

Les colonnes obligatoires sont contig, start et end. Les autres valeurs puvent-être remplacées par des .

Exemple :
COSV57675673 chr12 133249847 133249847 G A POLE c.1376C>T p.S459F
COSV57196518 chr5 1295228 1295228 G A TERT . .
COSV57196695 chr5 1295228 1295228 G T TERT . .
. chr2 29443642 29443642 C T ALK c.3575G>A p.R1192Q
. chr2 29443639 29443639 A T ALK c.3578T>A p.F1193Y
. chr2 29443629 29443631 . . ALK . .
. chr2 29443623 29443625 . . ALK . .
. chr15 98631837 90631839 . . IDH2 . .
. chr11 534285 534286 . . HRAS . .

Une fois un ensemble de hotspots ajouté à NGSDiagLim, il peut être désactivé à partir du tableau Liste des hotspots. La désactivation rendra son utilisation impossible lors de l'ajout d'analyses. Il peut être réactivé à tout moment.

Couvertures spécifiques

Dans les paramètres d'analyses principaux, il est possible de mettre 2 seuils de profondeur pour l'analyse de couverture (voir Création de paramètres d'analyse). Cependant, il est également possible de définir des profondeurs spéciales, spécifiques à certaines régions du panel.

Par exemple, il est possible de vérifier que toutes les régions sont couvertes à au moins 500X, et de définir une profondeur minimale de 3000X pour l'exon 14 du gène JAK2.

Pour rajouter des couvertures spécifiques, sélectionnez l'onglet Couvertures spécifiques. Cliquez ensuite sur le bouton Ajouter des couvertures spécifiques. Nommez l'ensemble des régions que vous allez définir. Ce nom doit être unique. Vous pouvez ensuite ajouter des régions, en définissant pour chacunes d'elles, leur nom, leur chromosome, leurs positions start et end, et la profondeur minimum requise.
Cliquer ensuite sur le bouton Ajouter la région pour ajouter cett région à l'ensemble de régions avec des couvertures spécifiques.
Lorsque toutes les régions ont été définies, cliquez sur le bouton OK, en bas de la fenêtre, pour valider l'ensemble de couvertures spécifiques.

Un ensemble de couvertures spécifiques peut être désactivé pour ne plus pouvoir l'utiliser. Il est possible de le réactiver à tout moment.

CIQs

Il est possible de définir des CIQs (échantillons contrôles) pour le suivi de certains variants entre les runs. Pour cela, sélectionnez l'onglet CIQs et cliquez sur le bouton Ajouter un CIQ

Un CIQ est défini par :

Un nom
Identifiant interne du CIQ. Doit être unique
Un barcode
Une expression régulière qui définie le barcode du CIQ. Cela sert à reconnaitre automatiquement le CIQ depuis l'importation d'un run. Si ce champ n'est pas rempli, les analyses CIQ ne seront pas reconnues automatiquement, mais il sera toujours possible des les définir comme "Analyse CIQ" lors de l'importation du run.
Exemple, pour un CIQ dont le barcode se nomme toujours de la façon suivante : CIQ-identifiant-LAM, l'expression régulière suivante peut-être utilisée pour reconnaître ce CIQ : CIQ.+LAM. (voir : Rules of writing regular expressions)
Une liste de variants
hotspots
Ces hotspots correspondent aux variants attendus du CIQ. Ils sont définis par un nom, un chromosome (contig), leurs positions start et end, l'allèle de référence, l'allèle alternatif et enfin la VAF ciblée. C'est la VAF attendue du hotspot.
Renseignez ces différents champs et cliquez sur le bouton Ajouter le hotspot pour ajouter le variant à la liste des variants du CIQ.

Validez ensuite le CIQ en cliquant sur le bouton OK en bas de la fenêtre. Il est possible de désactiver un CIQ et de le réactiver à tout moment.

Création de paramètres d'analyse

Lorsqu'au moins un panel et un ensemble de gènes et transcripts ont été ajoutés à NGSDiagLim, la création de paramètres d'analyse peut être effectuée.

Il s'agit de définir les paramètres à utiliser pour importer une analyse. A partir du module de gestion des paramètres d'analyses, sélectionnez l'onglet Création de paramètres d'analyses. Donnez un nom unique à ces paramètres et sélectionnez le génome de référence à utiliser (GRCh37 ou GRCh38).

Choisir une profondeur minium. Toutes les régions en dessous de cette profondeur seront considérées comme non-couverte lors de l'analyse de couverture.

Choisir une profondeur d'alerte. Les régions avec une profondeur inférieure à cette valeur et supérieure à la profondeur minimale seront considérées comme couvertes, mais un warning leur sera appliquées lors de l'analyse de couverture.

Ces valeurs ne sont pas utilisées pour filtrer les variants. Si un variant est présent dans le vcf à importer avec une profondeur inférieure à la profondeur minimum, il sera quand même visible.

Choisir une VAF minimum (VAF = Nombre de reads supportant l'allèle alternatif / Profondeur totale à cette position).

La variants dont la VAF est inférieure à cette valeur seront filtrés et ne seront pas importés, sauf s'il correspondent à un hotspot.

Sélectionner le Panel ainsi que les gènes à utiliser.

Vous pouvez définir des régions à couvertures spécifiques ainsi qu'une liste de hotspots à utiliser si vous en avez défini.

Enfin, sélectionnez la méthode d'enrichissement de la librairie (Capture ou amplicon).

Une fois des paramètres d'analyse ajoutés à NGSDiagLim, ils peuvent être désactivés à partir du tableau Liste des paramètres d'analyse. Leur désactivation rendra leur utilisation impossible lors de l'ajout d'analyses. Ils peuvent être réactivés à tout moment. Des paramètres peuvent aussi être supprimés à la seule condition qu'aucune analyse ne soit déjà importée en ayant utilisé ces paramètres.

Importation d'analyses

Création d'un run

Chaque analyse doit être rattachée à un run. Un run correspond donc à un ensemble d'analyses qui sont typiquement les patients qui ont été séquencées en même temps.

Pour créer un run, à partir de la page d'accueil, cliquez sur le bouton Ajouter un run. Dans la fenêtre qui s'ouvre, renseignez le nom du run (doit être unique) et la date du run. Cliquez ensuite sur le bouton OK pour valider la création du run. Le run créé apparaîtra dans le tableau Liste des runs de la page d'accueil

Ajout d'analyses

Pour ajouter des analyses à un run, depuis la page d'accueil sélectionnez le run souhaité dans le tableau Liste des runs. Ensuite cliquez sur le bouton Ajouter des analyses au dessus du tableau Liste des analyses.

Fichiers d'entrée

Une analyse est au minimum constituée d'un fichier VCF. Ce fichier VCF doit être annoté grâce à Ensembl VEP.

Il est fortement conseillé d'utiliser le plugin dbNSFP de vep pour avoir un maximum d'informations sur les variants.

LEs variants doivent être normalisé avec bcftools :
bcftools norm -f /path/to/reference.fasta -m -both -o outfile.vcf.gz -O z input.vcf

Associé à ce vcf, il est possible de charger un fichier BAM et un fichier Depth dont le nom doit se terminer par _depth.tsv (ou sa version gzippé. _depth.tsv.gz)

Ce fichier Depth est le résultat de la commande samtools depth et qui contient pour chaque position ciblée, la profondeur de séquençage. Ce fichier, s'il est présent sera utilisé pour l'analyse de couverture. S'il n'est pas présent, c'est le fichier BAM qui sera utilisé. Dans le cas de librairie par capture, ce fichier depth sera également utilisé pour la détection de CNVs avec l'algorithme de CovCop.

Lors de l'ajout d'analyses dans un run, certains fichiers peuvent être rattachés au run. Ils peuvent correspondre par exemple les rapports d'analyse ou de séquençage. Les fichiers automatiquement importés sont les fichiers avec les extensions suivantes :

.pdf, .html, .xls, .xlsx, .txt, .tsv, .csv

L'ajout d'analyses peut se faire en sélectionnant un dossier complet de résultats d'analyses en cliquant sur le bouton Importation à partir d'un dossier. NGSDiagLim va détecter les différents fichiers et les associer en analyses automatiquement. Il est possible de rajouter individuellement une analyse en cliquant sur le bouton Importer une analyse et en sélectionnant un fichier VCF à importer.

Lorsque plusieurs analyses sont prêtes à être importées, elles apparaissent dans le tableau Liste des analyses. Pour chaque analyse, il faut obligatoirement sélectionner les paramètres à utiliser pour l'importer. Il est possible de définir une analyse comme étant un CIQ.

Plusieurs erreurs peuvent apparaître lors de l'importation d'analyses. Ces erreurs sont symbolisées par un triangle rouge, dans la première colonne du tableau. Une infobulle explique l'erreur. La liste des erreurs possibles sont :

Les paramètres d'analyse sont manquants
Dans la colonne Paramètre d'analyse, il faut sélectionner les paramètres d'analyse à utiliser pour son importation.

Le fichier vcf est manquant
Aucun fichier VCF n'est associé à l'analyse à importer. Dans la colonne Vcf, cliquez sur le bouton Charger un fichier et sélectionner le fichier VCF à importer.

Une analyse du même nom existe déjà
Deux analyses d'un même run ne peuvent pas avoir le même nom. Il faut renommer une des deux analyses. Dans la colonne Nom d'analyse faites, un double clic sur le nom de l'analyse en double, donnez un nouveau nom à l'analyse et validez le nom en appuyant sur la touche Entrée.

Le nom de l'échantillon est manquant
Chaque analyse doit être associée à un nom d'échantillon. En général cela correspond à l'identifiant interne du patient et/ou son barcode lors de la préparation de la librairie.

Le nom de l'analyse doit contenir seulement des caractères alphanumériques et les caractères - ou _
Le nom des analyses ne peuvent pas contenir de caractère spéciaux. Les caractères autorisés sont les caractères alphanumériques ([a-Z], et [0-9]) ainsi que les caractères "-" et "_". Les autres caractères (espace, ".", ",", ";" etc) sont interdits.

Cliquez sur le bouton OK en bas de la fenêtre pour importer les analyses. Les analyses qui contiennent des erreurs ne seront pas importées.

Suppression d'analyses

Les analyses importées peuvent être supprimée en cliquant sur le bouton Supprimer l'analyse dans le tableau Liste des analyses. De la même façon, un run entier peut être supprimé en cliquant sur le bouton Supprimer le run dans le tableau Liste des runs.

Interprétation d'une analyse

L'ouverture d'une analyse se réalise en cliquant sur le bouton Ouvrir l'analyse dans le tableau Liste des analyses ou directement en double-cliquant sur une ligne de ce tableau.

Plusieurs modules sont disponibles :

Variants

C'est le module principal. Il contient la liste des variants (SNP et Indels) qui ont été retrouvés chez le patient durant le variant calling. Il est composé d'un tableau contenant l'ensemble des variants, et d'un panneau en bas de page qui contient le détail du variant sélectionné.

Panel de gènes

Il est possible de définir des panels de gènes "virtuels" qui permettent de ne voir les variants positionnés seulement sur une liste de gènes d'intérêt. Pour ça, cliquez sur le bouton Créer un panel. Dans la fenêtre qui s'ouvre, donnez un nom à ce panel. Il est possible de charger un fichier texte avec le nom des gènes d'intéret, soit de les sélectionner directement dans la liste des gènes proposés. Une fois créé, le panel est disponible dans le menu déroulant au dessus du tableau des variants.

Colonnes visibles

Le tableau des variants possède un grand nombre de colonnes, certaines visibles par défaut, d'autres non. Il est possible de changer les colonnes visible en cliquant sur le bouton Colonnes visibles et en sélectionnant les colonnes d'intéret. L'ordre des colonnes peut également être redéfini en cliquant sur leur entête et en les faisant glisser tout en maintenant le clic appuyé.

Sélection du transcript

Lorsqu'un variant est annoté sur plusieurs transcripts, il est possible de choisir quel transcript utiliser. Dans la colonne Transcript du tableau des variants, cliquer sur l'icone menu devant le nom d'un transcript et sélectionnez le transcript à utiliser. Les transcripts préférés sont identifiés par le suffixe "(pref)". Si dans le tableau, un variant est visualisé sur un transcript qui n'est pas le transcript préféré, le nom du transcript apparait alors en rouge.

Ajout de commentaires au variant

Il est possible, à partir du panneau de détail du variant, d'ajouter un commentaire. Ce commentaire peut être général au variant, c'est-à-dire que si le variant est retrouvé dans une autre analyse, le commentaire sera visible. Il est aussi possible de mettre un commentaire sur le variant mais spécifique à l'analyse. Le commentaire n'apparaitra pas dans une analyse tierce.

Il est possible de ne visualiser seulement ses propres commentaires. Les commentaires des autres utilisateurs seront cachés.

Pathogénicité des variants

Pour changer la pathogénicité d'un variant, cliquez sur l'icône pathogénicité dans le panneau de détail du variant. Une fenêtre s'ouvre, et vous pouvez sélectionner la nouvelle pathogénicité parmi les 5 choix suivant :

  • Pathogenic
  • Likely pathogenic
  • Uncertain significance
  • Likely benign
  • Benign

Pour valider la pathogénicité, il est nécessaire de laisser un commentaire. Chaque changement de pathogénicité est sauvegardé et l'historique du variant est disponible depuis cette même fenêtre.

Variant Faux-Positif

Il est possible de définir un variant comme étant "Faux positif". Les variants faux positifs apparaitront à la fin de tableau, en gris. Cliquez sur l'icône Faux positif dans le panneau de détail du variant. Une fenêtre s'ouvre, et vous pouvez indiquer si le variant est un faux positif ou non.

Comme pour le changement de pathogénicité, il est nécessaire de laisser un commentaire pour valider chaque changement. L'historique du variant est disponible depuis cette même fenêtre.

Vérification Sanger

NGSDiagLim permet d'indiquer si un variant est en train ou a été validé par méthode Sanger. Cliquer sur l'icone confirmation sanger dans le panneau de détail du variant. Une fenêtre apparaît et il est possible de changer l'état du variant parmi ces 5 possibilité :

  • Aucun
  • Demandée
  • En cours
  • Terminée - négatif
  • Terminée - positif

Il est nécessaire de laisser un commentaire pour valider chaque changement. L'historique du variant est disponible depuis cette même fenêtre.

CNV

Importation des données

Il est possible de réaliser une analyse de CNV depuis NGSDiagLim selon l'algorithme de CovCop. Deux algorithmes existent selon la méthode de préparation de la librairie :

CovCop Amplicon

Cette méthode demande de charger une matrice de profondeur dont la première colonne correspond au gène et la seconde colonne à l'identifiant de la région. Ensuite, il y a une colonne par patient avec le nombre de reads alignés sur chacune des régions.

C'est le format de fichier rendu par le plugin Coverage de la TorrentSuite.

Exemple :

Gene Target IonXpress_063 IonXpress_064 IonXpress_065
PLEKHG5 AMPL7153475646 1884 1349 1456
PLEKHG5 AMPL7153160806 10084 5522 7378
PLEKHG5 AMPL7154498024 4527 2044 2358

Cette matrice doit correspondre au Panel utilisé pour réaliser les analyses. L'identifiant de chaque région de la matrice doit être identique à une région du panel utilisé.

CovCop Capture

Dans le cas d'une librairie par capture, NGSDiagLim va utiliser une fenêtre glissante pour analyser les régions séquencées. Il est impératif que les analyses à analyser ont été importées avec un fichier "depth.tsv".

Le choix de la taille de la fenêtre glissante peut être modifié et doit être adapté à chaque librairie selon la taille des fragments séquencés et la longueur des reads. Une fenêtre de 150 à 200pb est un bon début pour une première analyse.

Données brutes

Cette page affiche les données importées brutes. Si l'algorithme choisi est Amplicon, cela correspond à la matrice importée. Dans le cas de la méthode Capture, c'est une matrice contenant toutes les profondeurs calculées chez chacun des patients et pour chaque fenetre glissante.

CovCop Contrôles

Le principe de base de l'algorithme CovCop est de comparer chacun des patients d'un run, aux autres patients du run. Cependant, il est possible de définir soit-même des contrôles à utiliser comme référence. Après avoir chargeé des données (soit une matrice, soit les fichiers depth), cliquer sur le bouton Définir des contrôles.

Patients du run
Ce sont des contrôles internes. Sélectionner les patients du run qui seront utilisés comme contrôles.
Groupe de contrôles

Ce sont des contrôles externes, c'est-à-dire des patients qui ont été séquencés dans un autre run, et rassemblés dans un groupe de contrôles. Voir : Ajouter un groupe de contrôles

Il est également possible de définir un patient contrôle depuis l'entête du tableau.

Détection du sexe des patients

Il est possible de choisir le sexe de chaque patient en cliquant sur le bouton XY ou XX dans l'entête des colonnes du tableau.

Si certaines régions sont positionnées sur le chromosome X, la normalisation des données sera plus efficace en renseignant correctement le sexe de chacun des patients. Si des régions sont positionnées sur le chrY, la détection du sexe est automatique.

Contrôle qualité

La page Contrôle qualité permet de vérifier la qualité des données importées. Elle permet de visualiser si la profondeur de séquençage est homogène entre les patients et entre les différents pools d'amplification dans le cas d'une librairie de type amplicon.

Il est possible de supprimer les données d'un patient. Sélectionnez le patient dans le graphique du haut (Profondeur de séquençage par patient). Le graphique Profondeur de séquençage par pool montre alors la profondeur d'amplification entre le différents pools du patient sélectionné. Dans le cas d'une libraire enrichie par capture, seul un pool nommé A est présent.
Il est possible de supprimer un ou l'ensemble des pools du patient en cliquant sur le bouton Supprimer le pool et Supprimer tous les pools respectivement.

Détection des CNVs

Après avoir cliqué sur le bouton Normaliser les données, deux nouvelles pages sont accessibles : Données normalisées et Carte des CNVs. Ces deux pages possèdent un même encart qui permet de gérer les paramètre de détection des CNVs :

Mode automatique

Le mode automatique de détection est activable en sélectionnant le bouton Auto. Ce mode se base sur le z-score pour déterminer automatiquent les seuil de délétion et duplication.

Mode manuel

Le mode manuel est accessible en déselectionant le bouton Auto. Il permet de fixer manuellement les seuils de déletion et duplication à l'aide du curseur positionné à côté.

Pour une analyse de recherche de CNVs constitutionels, les seuils de déletion/duplication de 0.7/1.3 ou 0.6/1.4 sont de bons point de départ à l'analyse.
Dans le cas d'une analyse somatique, il peut être intéressant de resserer les seuils à 0.8/1.2, mais en prenant le risque d'appeler des faux positifs.

Nombre de région par CNV

Comme dans le cas d'une CGH array, covcop attends qu'une succession de région indique un signal de déletion ou duplication pour appeler un CNV. Le nombre minimum de régions successives est fixé par défaut à 3. Il peut être changé par l'utilisateur en utilisant le champ : Nombre de régions minimum par CNVs

Chaque changement de paramètre (mode automatique/manuel, changement de seuil de délétion/duplication et nombre de régions successives minimum) implique une nouvelle recherche de CNVs en cliquant sur le bouton Chercher les CNVs.

Données normalisées

La page Données normalisées contient un tableau montrant les valeurs normalisées pour chaque patient et chacune des régions analysées. Les régions correspondent à un amplicon dans le cas d'un enrichissement par amplicon ou à une fenêtre glissante dans le cas d'un enrichissement par capture.

Les régions détectées comme délétées sont de couleurs orange, tandis que les régions dupliquées apparaissent en rouge.

Carte des CNVs

La page Carte des CNVs représente pour chaque patient la valeur des régions normalisées sous la forme d'un graphique.

Sur chaque graphique, l'axe verticale correspond à la valeur normalisée, et l'axe horizontal à la position chromosomique. Les régions sont triées selon leur positions chromosomiques.

Un point de couleur rouge indique une région dupliquée tandis qu'un point de couleur orange indique une région délétée. Les CNVs sont visualisés par des rectanges oranges et rouges, indiquant des délétions et duplications respectivement.

Courbe de Loess

Il est possible de visualiser la courbe de régression locale (Loess) en cliquant sur le bouton Loess.

Moyenne des gènes

La valeur normalisée moyenne des gènes peut être affichée en cliquant sur le bouton Moyennes des gènes.

Cumulative summary

Des courbes de type cumulative summary peuvent être affichées en cliquant sur le bouton Courbes Cusum. Ce sont des courbes qui indiquent des déviations par rapport à la moyenne. Ces déviations peuvent être positives (courbes bleues) ou négative (courbes roses). Elles peuvent être utiles pour visualiser des CNVs à très faible VAF, dans le cas d'analyse somatique ou de mosaïque.

Navigation dans les graphiques

Il est possible de zoomer sur certaines zonne grâce à la molette de la souris ou en cliquant avec le bouton gauche de la souris et en définissant une zone en gardant le clic appuyé. Un clic droit sur un graphique permet de réinitaliser le zoom. Il est possible de se déplacer sur le graphique, en cliquant et laissant appuyé le bouton molette de la souris.

Chaque graphique peut être exporté, soit vers un fichier .png soit directement en le copiant dans le presse papier grâce aux boutons au dessus de chaque graphique.

Ajouter un groupe de controles

Il est possible de définir des groupes de contrôles externes pour effectuer les analyses de CNVs. Les patients seront donc comparés à ce groupe au lieu d'être comparés les uns par rapport aux autres.

Cliquez sur le menu Administrateur > Gestion des paramètres des CNVs. Cliquez ensuite sur le bouton Ajouter un groupe contrôle

Donnez un nom unique à ce groupe et sélectionnez le panel auquel il correspond. Ensuite, deux cas de figure sont possibles :

Amplicon

Il faut sélectionner une matrice de couverture identique à celle utilisée pour une analyse de CNV amplicon (CNV amplicon) où chaque colonne de patient correspond à un contrôle.

Capture

Il faut charger un ensemble de fichiers "depth" à utiliser comme contrôles. Les positions présentent dans ces fichiers doivent correspondre aux régions du panel.

Couverture

Si l'analyse importée contenait un fichier bam ou un fichier depth, l'analyse de couverture a pu être réalisée. C'est une analyse qui détecte les régions non couvertes ou mal couvertes en fonction critères définis dans les paramètres d'analyses.

Le résultat est sous la forme d'un tableau qui indique les régions détectées, avec leurs tailles, leurs profondeurs moyennes et les gènes impliqués Il est possible de visualiser ces régions directement dans IGV en cliquant sur le bouton "IGV"

IGV doit absolument être lancé par l'utilisateur pour pouvoir être utilisé.

Deux types d'analyses peuvent être réalisées : Analyse de couverture générale ou spécifique si des régions spécifiques ont été définies dans les paramètres d'analyses.

Images et Commentaires

Chaque utilisateur peut ajouter un commentaire à une analyse ou importer une image. Sélectionnez l'onglet Informations additionnelles et cliquer sur le bouton Ajouter un commentaire. Une fenêtre apparait vous permettant d'écrire un texte qui sera rattaché à l'analyse.
Le bouton Importer une image permet d'attacher une image à l'analyse.

CIQ

Si l'analyse visualisée est de type CIQ, le suivi des hotspots est disponible dans l'onglet CIQ.

La liste des hotspots est indiquée dans le tableau Hotspots du CIQ. En sélectionnant un hotspot, il est possible d'obtenir ses valeurs ainsi que la liste des précédentes mesures

Pour chaque hotspot, est calculé la moyenne et l'écart-type des VAF sur les précédentes mesures acceptées, ainsi que les intervalles de confiance à ±2 et ±3 écarts-types.

Il est possible d'accepter ou de rejeter une mesure. Lors du changement d'état d'une mesure (rejetée=>acceptée; acceptée=>rejetée), un commentaire peut être laissé par l'utilisateur. Il est possible de visualiser l'historique d'une mesure en cliquant sur le bouton dans la colonne Acceptation du point du tableau Liste des mesures. Seules les mesures acceptées seront prises en compte pour le calcul des moyennes, écarts-types et intervalles de confiance.

Une mesure supérieure à 3 écarts-types apparaitra en rouge, et en orange si elle est supérieure à 2 écarts-types.

Navigation dans le graphique

Il est possible de zoomer dans le graphique des mesures grâce à la molette de la souris.

En maintenant la touche ctrl appuyée, un zoom vertical sera effectué. A l'inverse, un zoom horizontal est possible en appuyant sur la touche alt

Un clic droit réinitalisera le zoom.

Meta données

L'onglet Meta données contient toutes les informations sur les conditions de l'analyse. Il est possible de voir quel utilisateur à importé l'analyse, à quelle date, les fichiers qui ont été importés etc...

Le détail des paramètres utilisés pour l'analyse est également visible.

L'entête du fichier vcf utilisé est affiché. Cet entête peut contenir davantage d'informations sur l'analyse bioinformatique qui a été menée.

Il est possible de chercher directement un variant dans le fichier vcf grace au formulaire de recherche au-dessus de l'entête.

Exemples :

chr3-20124458 : Recherche tous les variants à cette position
chr7-54789652-A-T : Recherche spécifiquement ce variant
chr6-21547841-21547852 : Recherche tous les variants qui overlapent cette région

Run info

L'onglet "Run info" contient les informations du run. Il est possible de voir quel utilisateur a créé le run et à quelle date.

Les fichiers rattachés au run qui ont été importés lors de l'ajout d'analyses peuvent également être ouverts directement depuis cet onglet.

Informations techniques

Installation

Télécharger l'archive : https://path/to/release et l'extraire dans son dossier d'installation. Exécutez le fichier exec.bat pour lancer l'application.

Architecture

Java

NGSDiagLim nécéssite java>=17 et javafx>=17. Ces dépendances sont embarquée directement dans l'archive.

Base de données

NGSDiagLim utilise une base de donnée embarquées h2. Cette base de données est automatiquement créé dans le dossier $installdir/database est porte le nom NGSDiaglim

Données importées

NGSDiagLim nécéssite certains fichiers pour son fionctionnement. Lors de l'importation d'analyses, il copie également les fichiers nécéssaires. Ces différents fichiers se trouvent dans le répertoire $install_dir/.data

.data/reference

C'est un dossier qui contient les différentes séquences de référence du génome humain (GRCh37 et GRCh38), ainsi que leurs indexes.

.data/panels

Ce dossier contient les fichiers .bed gzippés de chaque panel importé dans NGSDiagLim.

.data/runs_data

Ce dossier contient l'ensemble des fichiers copiés lors de l'importation des analyses. Ce sont ces fichiers qui seront ensuite lus pour charger les données.

Il y a un dossier par année, exemple : .data/runs_data/2022/. Dans chaque dossier année, il y a un dossier pour chaque run créé.

Chaque dossier run contient ensuite 2 sous dossiers :

analyses
Contient un dossier pour chaque analyse importée de ce run. Le contenu de ce dossier est décris plus bas.
runs_files

Contient tous les fichiers relatifs au run. Ce sont par exemple les fichiers de rapport d'analyse.

Dossier d'analyse

Dans le dossier analyses de chaque run, se trouve un sous dossier pour chacune des analyses. Ces sous dossier contiennt à leur tour un ensemble de fichier qui seront lu et chargé lors de l'ouverture d'une analyse sous NGSDiagLim.

analyses
Contient un dossier pour chaque analyse importée de ce run. Le contenu de ce dossier est décris plus bas.
variant.vcf.gz et variant.vcf.gz

Fichier .vcf gzippé et indexé avec tabix, qui contient tous les variants retrouvés chez le patient ou l'échantillon. C'est ce fichier vcf qui est parsé lors de l'ouverture de l'analyse.

coverage.bed.gz

Ce fichier est présent seulement si lors de l'importation de l'analyse, l'utilisateur a rattaché soit un fichier bam, soit un fichier depth. (Voir Ajouter des analyses).

C'est un fichier .bed gzippé qui contient les régiions mal couvertes ou non couvertes. Ce fichier bed contient des tag spécifiques à IGV pour configurer la track lors de son importantion dans IGV.

*_depth.tsv.gz

Ce fichier est présent seulement si lors de l'importation de l'analyse, l'utilisateur a rattaché un fichier depth. (Voir Ajouter des analyses).

C'est un fichier .tsv gzippé qui correspond à la sortie de la commande samtools depth. Ce fichier est utilisé pour appler les CNVs par l'algorithme de CovCop capture.

Pour des raison d'espace disque, les fichiers bam ne sont pas copiés dans le répertoire .data.

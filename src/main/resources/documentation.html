Introduction

NGSDiag est un outil d'aide à l'interprétation de variants génétiques (SNP/Indels/CNVs)

Gestion des utilisateurs

Le module de gestion des utilisateurs est accessible via le menu Administrateur > Gestion des utilisateurs. Seuls les utilisateurs ayant la permission de gérer les autres utilisateurs ont accès à ce menu.

Groupes d'utilisateurs

Chaque utilisateur appartient à un ou plusieurs groupes. Ces groupes contiennent une liste de permissions, c'est-à-dire d'actions à pouvoir réaliser au sein de NGSDiag.

Deux groupes sont initialement créés :

  • ADMIN : possède tous les droits
  • GUEST : ne possède aucun droit (accès en lecture seulement)

Ces deux groupes ne peuvent pas être supprimés

L'ajout d'un nouveau groupe se fait en cliquant sur le bouton "Ajouter un groupe" et en lui donnant un nom unique dans la fenêtre qui apparaît.

Pour définir les permissions de ce groupe, il suffit de sélectionner le groupe dans le tableau "Liste des groupes" et de choisir ensuite les permissions qui lui seront associées.

Ajouter/Editer un utilisateur

Lors de la première initialisation de NGSDiag, un utilisateur administrateur est créé. (login/mot de passe : admin/admin

Il est très fortement recommandé de désactiver cet utilisateur ou de changer son mot de passe

Pour ajouter un nouvel utilisateur, cliquer sur le bouton "Ajouter un utilisateur". Une fenêtre apparaît. Renseigner l'identifiant qui doit être unique et son mot de passe. Choisir au moins un groupe auquel rattaché l'utilisateur.

Le mot de passe d'un utilisateur peut être réinitialisé en cliquant sur l'icône "reset" dans le tableau "Gestion des comptes utilisateurs".

Un utilisateur peut être désactivé, c'est-à-dire qu'il sera impossible de se connecter avec son compte, mais il ne sera pas supprimé de la base de donné. À tout moment un compte désactivé peut être réactivé.

Les groupes de chaque utilisateur peuvent aussi être modifiés à tout moment.

Paramètres d'analyse

Chaque analyse qui sera importée devra être rattaché à certains paramètres. Le module de gestion de ces paramètres est accessible via le menu Administrateur > Gestion des paramètres d'analyse.

Panel

Le panel correspond aux régions ciblées lors de la préparation de la librairie (amplicon ou capture). A partir du module de gestion des paramètres d'analyses, sélectionner l'onglet Gestion des panels. Pour ajouter un panel cliquer sur le bouton "Ajouter un panel". Dans la fenêtre qui s'ouvre, renseigner le nom du panel à rajouter (doit être unique) et cliquer sur le bouton "Ouvrir un fichier bed" pour sélectionner le fichier bed qui contient les régions séquencées.

C'est un fichier qui doit contenir au moins les 4 colonnes suivantes :

Contig
nom du chromosome
Start
Position du début de la région
End
Position de fin de la région
Name
Nom de la région, par exemple le gène ciblé. Ce nom ne doit pas forcément être unique à chaque région.

Exemple :
chr1 6557013 6557096 PLEKHG5
chr1 6557374 6557388 PLEKHG5
chr1 6579499 6579576 PLEKHG5
chr1 10292381 10292497 KIF1B
chr1 10316299 10316386 KIF1B
chr1 10318545 10318735 KIF1B

Une fois un panel ajouté à NGSDiag, il peut être désactivé à partir du tableau "Liste des panels". La désactivation d'un panel rendra son utilisation impossible lors de l'ajout d'analyses. Un panel peut être réactivé à tout moment.

Gènes et transcripts

Les gènes et transcript correspondent aux noms des gènes et aux transcripts à utiliser pour l'annotation des variants. A partir du module de gestion des paramètres d'analyses, sélectionner l'onglet Gènes et Transcripts. Pour ajouter un ensemble de gènes et leurs transcripts associés, cliquer sur le bouton "Ajouter des gènes/transcripts. Dans la fenêtre qui s'ouvre, renseigner le nom du panel à rajouter (doit être unique) et cliquer sur le bouton "Ouvrir le fichier" pour charger les gènes et transcripts.

C'est un fichier qui doit contenir les 2 colonnes suivantes :

Gene
Nom du chromosome
Transcripts
Liste des transcripts à utiliser. Il peut y avoir plusieurs transcripts pour un gène. Il faut séparer leur id par le caractère : ;

Exemple :
ALPL NM_000478
ANK3 NM_001204403;NM_001204404;NM_020987;NM_001149
AP2S1 NM_001301078;NM_001301076;NM_001301081
ATL1 NM_001127713

Si un gène possède plusieurs transcripts possible, il est possible de choisir le transcript à utiliser en priorité en cliquant dessus. C'est ce transcript qui sera utilisé par défaut pour l'affichage des variants du gène. Par defaut, si un gène ne possède qu'un seul transcript, il est automatiquement sélectionné comme transcript préféré.

Une fois un ensemble de gène et transcripts ajouté à NGSDiag, il peut être désactivé à partir du tableau "Liste des gènes/transcripts". La désactivation rendra son utilisation impossible lors de l'ajout d'analyses. Il peut être réactivé à tout moment.

Hotspots

Chaque analyse peut être associée à un sensemble de hotspot. Ce sont une liste de variants ou de position qui seront systématiquement gardées lors de l'importation d'un VCF. Aucun filtre ne sera appiqué sur ces variants.

Pour rajouter des hotspots, à partir du module de gestion des paramètres d'analyses, sélectionner l'onglet Hotspots. Cliquer ensuite sur le bouton "Ajouter des Hotspots". Dans la fenêtre qui s'ouvre, renseigner le nom de l'ensemble de hotspots à rajouter (doit être unique) et cliquer sur le bouton "Ouvrir le fichier" pour sélectionner le fichier qui contient les hotspots à importer.

C'est un fichier qui doit contenir les 9 colonnes suivantes :

ID
Identifiant du hotspot (ex son numéro cosmic)
Contig
Chromosome du hotspot
Start
Position de début du hotspot
End
Position de fin du hotspot
Ref
Allèle de référence
Alt
Allèle alternatif
Gene
Nom du gène
hgvsc
Nomenclature hgvsc
hgvsp
Nomenclature hgvsp

Les colonnes obligatoires sont contig, start et end. Les autres valeurs puvent-être remplacées par des .

Exemple :
COSV57675673 chr12 133249847 133249847 G A POLE c.1376C>T p.S459F
COSV57196518 chr5 1295228 1295228 G A TERT . .
COSV57196695 chr5 1295228 1295228 G T TERT . .
. chr2 29443642 29443642 C T ALK c.3575G>A p.R1192Q
. chr2 29443639 29443639 A T ALK c.3578T>A p.F1193Y
. chr2 29443629 29443631 . . ALK . .
. chr2 29443623 29443625 . . ALK . .
. chr15 98631837 90631839 . . IDH2 . .
. chr11 534285 534286 . . HRAS . .

Une fois un ensemble de hotspots ajouté à NGSDiag, il peut être désactivé à partir du tableau "Liste des hotspots". La désactivation rendra son utilisation impossible lors de l'ajout d'analyses. Il peut être réactivé à tout moment.

Création de paramètres d'analyse

Lorsqu'au moins un panel et un ensemble de gènes et transcripts ont été ajoutés à NGSDiag, la création de paramètres d'analyse peut être effectuée.

Il s'agit de définir les paramètres à utiliser pour importer une analyse. A partir du module de gestion des paramètres d'analyses, sélectionner l'onglet Création de paramètres d'analyses. Donner un nom unique à ces paramètres et sélectionner le génome de référence à utiliser.

Choisir une profondeur minium. Toutes les régions en dessous de cette profondeur seront considéré comme non-couverte lors de l'analyse de couverture.

Choisir une profondeur d'alerte. Les régions avec une profondeur inférieure à cette valeur et supérieure à la profondeur minimale seront considérées comme couvertes, mais un warning leur sera appliqué lors de l'analyse de couverture.

Ces valeurs ne sont pas utilisées pour filtrer les variants. Si un variant est présent dans le vcf à importer avec une profondeur inférieure à la profondeur minimum, il sera quand même visible.

Choisir une VAF minimum (VAF = Nombre reads supportant l'allèle alternatif / Profondeur totale).

La variants dont la VAF est inférieure à cette valeur seront filtrés et ne seront pas importés, sauf s'il correspondent à un hotspot.

Une fois des paramètres d'analyse ajoutés à NGSDiag, ils peuvent être désactivés à partir du tableau "Liste des paramètres d'analyse". Leur désactivation rendra leur utilisation impossible lors de l'ajout d'analyses. Ils peuvent être réactivés à tout moment. Des paramètres peuvent aussi être supprimés à la seule condition qu'aucune analyse ne soit importée en utilisant ces paramètres.

Importation d'analyses

Création d'un run

Chaque analyse doit être rattaché à un run. Un run correspond donc à un ensemble d'analyses. Typiquement ce sont les patients qui ont été séquencées en même temps.

Pour créer un run, à partir de la page d'accueil, cliquer sur le bouton "Ajouter un run". Dans la fenêtre qui s'ouvre, renseigner le nom du run (doit être unique) et la date du run.

Ajout d'analyses

Pour ajouter des analyses à un run, depuis la page d'accueil sélectionner le run souhaité dans le tableau "Liste des runs". Ensuite cliquer sur le bouton "Ajouter des analyses" au dessus du tableau "Liste des analyses".

Fichiers d'entrée

Une analyse est au minimum constituée d'un fichier VCF. Ce fichier VCF peut être annoté grâce à Ensembl VEP.

Associé à ce vcf, on peut charger un fichier BAM et un fichier appelé "Depth" dont le nom doit se terminer par "_depth.tsv" et qui peut être gzippé. "_depth.tsv.gz"

Ce fichier Depth est le résultat de la commande samtools depth et qui contient pour chaque position ciblée, la profondeur de séquençage. Ce fichier, s'il est présent sera utilisé pour l'analyse de couverture. S'il n'est pas présent, c'est le fichier BAM qui sera utilisé. Dans le cas de librairie par capture, ce fichier depth sera également utilisé pour la détection de CNVs avec l'algorithme de CovCop.

Lors de l'ajout d'analyses dans un run, certains fichiers peuvent être rattachés au run. Les fichiers supportés sont les fichiers pdf et html. Ils peuvent correspondre par exemple les rapports d'analyse ou de séquençage.

L'ajout d'analyse peut se faire soit en sélectionnant un dossier, GSDiag va détecter les différents fichiers et les associer en analyses séparées automatiquement, soit en sélectionnant chacun des fichiers individuellement.

Suppression d'analyses

Les analyses importées peuvent être supprimée en cliquant sur le bouton "Supprimer l'analyse" dans le tableau "Liste des analyses". De la même façon, un run entier peut être supprimer en cliquant sur le bouton "Supprimer le run" dans le tableau "Liste des runs".

Interprétation d'une analyse

L'ouverture d'une analyse se réalise en cliquant sur le bouton "Ouvrir l'analyse" dans le tableau "Liste des analyses" ou directement en double-cliquant sur une ligne de ce tableau.

Plusieurs modules sont disponibles :

Variants

C'est le module principal. Il contient la liste des variants (SNP et Indels) qui ont été retrouvés chez le patient durant l'analyse bioinformatique. Il est composé d'un tableau contenant l'ensemble des variants, et d'un panneau en bas de page qui contient le détail du variant sélectionné.

Panel de gènes

Il est possible de définir des panels de gènes "virtuels" qui permettent de ne voir les variants positionné seulement sur une liste de gènes d'intérêt. Pour ça, cliquer sur le bouton "Créer un panel". Dans la fenêtre qui s'ouvre, donner un nom à ce panel. Il est possible de charger un fichier texte avec le nom des gènes d'intérets, soit de les sélectionner directement dans la liste des gènes proposés. Une fois créé, le panel est disponible dans le menu déroulant au dessus du tableau des variants.

Colonnes visibles

Le tableau des variants possède un grand nombre de colonnes, certaines visibles par défaut, d'autres non. Il est possible de changer les colonnes visible en cliquant sur le bouton "Colonnes visibles" et en sélectionnant les colonnes d'intéret. L'ordre des colonnes peut également être redéfini en cliquant sur leur entête et en les faisant glisser tout en maintenant le clic appuyé.

Sélection du transcript

Lorsqu'un variant est annoté sur plusieurs transcripts, il est possible de choisir quel transcript utiliser. Dans la colonne "Transcript" du tableau des variants, cliquer sur l'icone "menu" devant le nom d'un transcript et sélectionner le transcript à utiliser. Les transcripts préférés sont identifiés par le suffixe "(pref)". Si dans le tableau, un variant est visualisé sur un transcript qui n'est pas le transcript préféré, le nom du transcript apparait en jaune.

Ajout de commentaires au variant

Il est possible, à partir du panneau de détail du variant, d'ajouter un commentaire. Ce commentaire peut être général au variant, c'est-à-dire que si chez une autre analyse le même variant est retrouvé, le commentaire sera visible. Il est aussi possible de mettre un commentaire sur le variant mais spécifique à l'analyse. Le commentaire n'apparaitra pas dans une analyse tierce.

Pathogénicité des variants

Pour change la pathogénicité d'un variant, cliquer sur l'icône "pathogénicité" dans le panneau de détail du variant. Une fenêtre s'ouvre, et vous pouvez sélectionner la nouvelle pathogénicité parmi les 5 choix suivant :

  • Pathogenic
  • Likely pathogenic
  • Uncertain significance
  • Likely benign
  • Benign

Pour valider la pathogénicité, il est nécessaire de laisser un commentaire. Chaque changement de pathogénicité est sauvegardé et l'historique du variant est disponible depuis cette même fenêtre.

Variant Faux-Positif

Il est possible de définir un variant comme étant "Faux positif". Il apparaitra alors dans les derniers variants visualisés. Cliquer sur l'icône "Faux positif" dans le panneau de détail du variant. Une fenêtre s'ouvre, et vous pouvez indiquer si le variant est un faux positif ou non.

Comme pour le changement de pathogénicité, il est nécessaire de laisser un commentaire pour valider chaque changement. L'historique du variant est disponible depuis cette même fenêtre.

Vérification Sanger

NGSDiag permet d'indiquer si un variant est en train ou a été validé par méthode Sanger. Cliquer sur l'icone "confirmation sanger" dans le panneau de détail du variant. Une fenêtre apparaît et il est possible de changer l'état du variant parmi ces 5 possibilité :

  • Aucun
  • Demandée
  • En cours
  • Terminée - négatif
  • Terminée - positif

Il est nécessaire de laisser un commentaire pour valider chaque changement. L'historique du variant est disponible depuis cette même fenêtre.

CNV

Il est possible de réalyser une analyse de CNV depuis NGDDiag selon l'algorithme de CovCop. Deux algorithmes existent selon la méthode de préparation de la librairie :

CovCop Amplicon

Cette méthode demande de charger une matrice de profondeur dont la première colonne correspond au gène. La seconde colonne à l'identifiant de la région. Et ensuite une colonne par patient avec le nombre de reads alignés sur chacune des régions.

CovCop Capture

Dans le cas d'une librairie par capture, NGSDiag va utiliser une fenêtre glissante pour analyser les régions séquencées. Il est impératif que les analyses à analyser ont été importées avec un fichier "depth.tsv".

Le choix de la taille de la fenêtre glissante peut être modifié et doit être adapté à chaque librairie selon la taille des fragments séquencés et la longueur des reads. Pour une librairie fragmentée à 500pb et des reads 2*300pb, une fenêtre de 150 à 200pb est un bon début pour une première analyse.

CovCop Contrôles

Le principe de base de l'algorithme CovCop est de comparer chacun des patients d'un run, aux autres patients du run. Cependant, il est possible de définir soit-même des contrôles à utiliser comme référence. Après avoir chargeé des données (soit une matrice, soit les fichiers depth), cliquer sur le bouton Définir des contrôles.

Patients du run
Ce sont des contrôles internes. Sélectionner les patients du run qui seront utilisés comme contrôles.
Groupe de contrôles

Ce sont des contrôles externes, c'est-à-dire des patients qui ont été séquencés dans un autre run, et rassemblés dans un groupe de contrôles. Voir : Ajouter un groupe de contrôles

Cette matrice doit correspondre au Panel utilisé pour réaliser les analyses. L'identifiant de chaque région de la matrice doit être identique à une région du panel utilisé.

Couverture

Si l'analyse importée contenait un fichier bam ou un fichier depth, l'analyse de couverture a pu être réalisée. C'est une analyse qui détecte les régions non couvertes ou mal couvertes en fonction des paramètres d'analyses.

Le résultat est sous la forme d'un tableau qui indique les régions détectées, avec leurs tailles leurs profondeurs moyennes et les gènes impliqués Il est possible de visualiser ces régions directement dans IGV en cliquant sur le bouton "IGV"

IGV doit absolument être lancé par l'utilisateur pour pouvoir être utilisé.

Informations additionnelles

Chaque utilisateur peut ajouter un commentaire à une analyse ou importer une image. Sélectionner l'onglet "Informations additionnelles" et cliquer sur le bouton "Ajouter un commentaire". Une fenêtre apparait vous permettant d'écrire un texte qui sera rattaché à l'analyse.
Le bouton "Importer une image" permet d'attacher une image à l'analyse.

Meta données

L'onglet "Meta données" contient toutes les informations sur les conditions de l'analyse. Il est possible de voir quel utilisateur à importé l'analyse, à quelle date, les fichiers qui ont été importés etc...

Le détail des paramètres utilisés pour l'analyse est également visible.

L'entête du fichier vcf utilisé est affiché. Cet entête peut contenir davantage d'information sur l'analyse bioinformatique qui a été menée. Il est possible de chercher directement un variant dans le fichier vcf grace au formulaire de recherche au-dessus de l'entête.

Run info

L'onglet "Run info" contient les informations du run. Il est possible de voir quel utilisateur a créé le run et à quelle date.

Les fichiers rattachés au run qui ont été importés lors de l'jout d'analyses sont également disponibles depuis cet onglet.

Ajouter un groupe de controles

Il est possible de définir des groupes de contrôles pour effectuer les analyses de CNVs. Les patients seront donc comparés à ce groupe au lieu d'être comparés les uns par rapport aux autres..

Cliquer sur le menu Administrateur > Gestion des paramètres des CNVs. Cliquer ensuite sur le bouton Ajouter un groupe contrôle

Donner un nom unique à ce groupe et sélectionner le panel auquel il correspond. Ensuite, deux cas de figure sont possibles :

Amplicon

Il faut sélectionner une matrice de couverture identique à celle utilisée pour une analyse de CNV amplicon (CNV amplicon) où chaque colonne de patient correspond à un contrôle.

Capture

Il faut charger un ensemble de fichiers "depth" à utiliser comme contrôles. Les positions présentent dans ces fichiers doivent correspondre aux régions du panel.
